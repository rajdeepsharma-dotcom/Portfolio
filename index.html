Instructional Design

How I design for learning

Behind every course is a structured design process — from storyboards to scenario blueprints. Here's what my ID work looks like in practice.

🗂️
Storyboarding
Storyboard Design in Storyline
I build annotated slide-by-slide storyboards that map each screen to a learning objective, interaction type, and asset list — before a single trigger is written.
Slide layout with on-screen text, visuals, and audio script
Interaction notes: click states, layers, branching logic
Client-ready format for sign-off before development
🎯
Learning Objectives
Bloom's Taxonomy Alignment
Every module starts with performance-based objectives written at the right cognitive level — from recall for compliance to synthesis for problem-solving courses.
Objectives mapped to Bloom's six levels
Assessment items directly tied to each objective
Gap analysis between current and desired performance
🔀
Scenario Design
Branching Scenario Blueprints
I design branching decision trees using Cathy Moore's action-mapping approach — keeping scenarios grounded in real job tasks, not abstract theory.
Action map showing business goal → tasks → practice activities
Consequence-driven paths with realistic outcomes
Built in Storyline 360 with variables and triggers
📊
Assessment Design
Quiz & Evaluation Strategy
I design assessments that actually measure learning — not just recall. From formative check-ins to summative SCORM-tracked quizzes with retry logic and feedback layers.
Question types matched to objective complexity
Custom feedback for correct and incorrect responses
SCORM score reporting to Moodle LMS
